IR seat events

Path: rp4_debug/cpe11-afcs/seatEvents/
Where is status: occupied? Only on documents whose ID ends with _ir_occupied (field status = occupied). Tap-out simulation creates _ir_available docs with status: available — those are usually the newest in the list, so it can look like “everything is available”. Search or sort by ID for ir_occupied.
Parent doc rp4_debug/cpe11-afcs does not have a top-level “occupied” flag. When seated, open seat1 or seat2 and check occupantUserId (that means the seat is physically occupied).
After the Cloud Function runs, each seat event also gets functionResult (ir_occupied_applied / ir_available_applied).
